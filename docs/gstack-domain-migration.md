# gstack 技能领域迁移指南

## 技能概述

- **技能名称**：gstack
- **评分**：3.249
- **核心功能**：快速无头浏览器（headless Chromium），用于 QA 测试和网页验证
- **性能**：首次启动约 3 秒，后续命令约 100-200ms
- **会话保持**：30 分钟无操作后自动关闭，期间 cookies、登录会话、标签页保持
- **安全审查结果**：✅ 纯技术文档，无安全问题

---

## 核心能力

### 1. 自动化测试工作流

| 功能 | 命令示例 | 说明 |
|------|-----------|------|
| 导航页面 | `goto <url>` | 打开指定 URL |
| 截图 | `screenshot [path]` | 保存页面截图 |
| 元素交互 | `click <sel>` | 点击元素 |
| 表单填写 | `fill <sel> <val>` | 填写表单 |
| 状态验证 | `is visible <sel>` | 验证元素可见 |
| 响应式测试 | `responsive [prefix]` | 多设备截图对比 |

### 2. 快照系统（核心工具）

- **交互模式**：`snapshot -i` - 显示所有可交互元素
- **差异对比**：`snapshot -D` - 与上次快照对比，显示变化
- **标注截图**：`snapshot -a -o path.png` - 带红色标注框的截图
- **光标交互**：`snapshot -C` - 包含可点击的 div 元素

## 领域迁移方案

### 领域 1：科学教学

#### 应用场景

| 场景 | gstack 用法 | 价值 |
|--------|-------------|------|
| **在线课程验证** | `goto <course_url>` → `screenshot /tmp/course.png` | 自动截图验证课程页面是否正常显示 |
| **学习平台测试** | `goto <lms_url>` → `snapshot -i` → `is visible "#login-btn"` | 自动测试登录按钮是否可见 |
| **教学资源检查** | `goto <resource_url>` → `text` | 提取文本内容，验证资源是否可访问 |
| **多设备兼容性** | `responsive /tmp/course-layout` | 一键生成移动端、平板、桌面端截图 |
| **作业提交测试** | `goto <submit_url>` → `fill @e1 "答案"` → `click @e2` | 自动模拟提交作业 |

#### 实战案例：在线教学平台测试

```bash
# 1. 打开教学平台
$B goto https://teaching.school.edu.cn

# 2. 检查登录界面
$B snapshot -i
$B is visible "#username-input"
$B is visible "#password-input"
$B screenshot /tmp/login-page.png

# 3. 模拟登录
$B fill @e1 "teacher001"
$B fill @e2 "password123"
$B click @e3
$B snapshot -D  # 查看登录后的变化

# 4. 验证课程列表
$B is visible "#course-list"
$B screenshot /tmp/courses.png

# 5. 测试响应式布局
$B responsive /tmp/teaching-layout
```

#### 教学应用优势

| 优势 | 说明 |
|------|------|
| 📸 **自动截图** | 无需手动打开浏览器，批量生成教学页面截图 |
| 📱 **多设备验证** | 一键测试移动端、平板、桌面端显示效果 |
| ✅ **状态验证** | 自动检查登录按钮、课程列表等关键元素是否存在 |
| 🔄 **回归测试** | 每次更新后自动验证教学平台功能是否正常 |
| 📊 **对比分析** | 通过 `snapshot -D` 对比更新前后的页面差异 |

| **表单提交测试** | `fill @e1 "申请内容"` → `click @e2` | 自动填写并提交表单 |
| **网页内容监控** | `goto <url>` → `text` | 定期提取页面内容，检查是否有变更 |
| **系统部署验证** | `goto <prod_url>` → `console` | 检查生产环境是否有 JS 错误 |

#### 实战案例：采购申请流程测试

```bash
# 定义完整流程
echo '[
  ["goto","https://oa.company.edu.cn/procure"],
  ["snapshot","-i"],
  ["fill","@e1","办公用品采购"],
  ["fill","@e2","1000"],
  ["click","@e3"],
  ["snapshot","-D"],
  ["is","visible","#success-message"],
  ["screenshot","/tmp/procure-result.png"]
]' | $B chain
```

#### 总务应用优势

| 优势 | 说明 |
|------|------|
| 🚀 **流程自动化** | 一条 `chain` 命令完成整个申请流程 |
| 📸 **证据留存** | 自动截图，保留申请成功的证据 |
| ✅ **状态验证** | 自动检查成功提示是否出现 |
| 🔄 **批量测试** | 测试多种申请场景，提高测试效率 |
| 🐛 **快速定位问题** | 通过 `console` 和 `network` 检查 JS 错误和请求失败 |

## 关键工作流模式

### 模式 1：验证型工作流

```
1. 打开页面 → 2. 截图 → 3. 检查关键元素 → 4. 保存证据
```

**适用场景**：
- 教学平台功能验证
- 办公系统状态检查
- 部署后冒烟测试

### 模式 2：对比型工作流

```
1. 基线快照（snapshot -i）→ 2. 执行操作 → 3. 对比快照（snapshot -D）
```

**适用场景**：
- 功能测试前后对比
- 表单提交状态验证
- 页面变更检测

### 模式 3：链式工作流

```
使用 `chain` 命令一次性执行多步操作
```

**适用场景**：
- 完整的业务流程测试
- 长流程自动化执行
- 回归测试套件

## 高级技巧

### 1. Cookie 导入（测试需要登录的页面）

```bash
# 从 Chrome 导入 cookie
$B cookie-import-browser

# 或导入特定域
$B cookie-import-browser --domain .qq.com
```

**应用场景**：
- 测试需要登录的教学平台
- 验证总务系统的用户权限

### 2. 对话框自动处理

```bash
# 自动接受下一个对话框
$B dialog-accept

# 接受带输入的对话框
$B dialog-accept "确认删除"
```

**应用场景**：
- 测试删除确认弹窗
- 处理提示对话框

### 3. 页面对比

```bash
# 对比两个环境的差异
$B diff https://staging.school.edu.cn https://prod.school.edu.cn
```

**应用场景**：
- 测试环境和生产环境对比
- 验证部署是否完整

### 4. JS 执行

```bash
# 检查页面标题
$B js "document.title"

# 检查特定内容
$B js "document.querySelector('.notice')?.textContent"
```

**应用场景**：
- 验证动态内容
- 提取页面数据
