# 桌面文件监控与自动路由系统

## 系统说明

本系统自动监控 **F:\桌面** 文件夹的新文件，并根据文件内容或名称智能路由到 IMA 知识库和本地记忆。

## 工作流程

```
F:\桌面 新文件生成
       ↓
   检测新文件
       ↓
   分析文件内容/名称
       ↓
   智能路由判断
       ↓
┌────────┬────────┬────────┐
│IMA存储 │本地存储│双重存储│
└────────┴────────┴────────┘
       ↓         ↓         ↓
    IMA    memory.md   IMA+memory.md
   知识库    本地记忆    双重备份
```

## 路由规则

根据文件内容或文件名中的关键词，自动判断存储位置：

| 关键词 | 存储位置 | 知识库/文件 |
|-------|---------|------------|
| 电话/微信/QQ/邮箱/地址/合同/协议/报价/账期/付款/供应商/客户/厂家/合作方 | IMA | 供应商管理 |
| 产品/库存/SKU/面料/工艺/行业报告/趋势/竞品/分析 | IMA | 产品资料库 |
| 活动/促销/618/双11/方案 | IMA | 618活动方案 |
| 话术/客服/回复/FAQ | IMA | 客服话术 |
| 今天/明天/待办/计划 | 本地 | memory.md |
| 感觉/想法/灵感/思路 | 本地 | MEMORY.md |
| 重要/关键/最终方案/决策 | 双重 | IMA奁地搜 + 本地MEMORY.md |
| 其他 | 本地 | memory.md |

## 自动化配置

### 自动执行
- **运行频率**：每小时执行一次
- **监控路径**：`F:\桌面`
- **工作目录**：`c:\Users\Administrator\WorkBuddy\Claw`
- **状态**：ACTIVE

### 手动触发
如果需要立即检查桌面新文件，运行：
```powershell
powershell -ExecutionPolicy Bypass -File "c:\Users\Administrator\WorkBuddy\Claw\scripts\monitor_desktop.ps1"
```

## 处理记录

### 已处理文件日志
- 位置：`c:\Users\Administrator\WorkBuddy\Claw\.codebuddy\automations\desktop-monitor\processed_files.txt`
- 用途：记录已处理的文件，避免重复处理

### 处理结果日志
- 位置：`c:\Users\Administrator\WorkBuddy\Claw\.codebuddy\automations\automation\memory.md`
- 内容：记录每个文件的处理结果、路由位置、时间戳

## 记忆引用方式

后续工作时，您可以这样引用：

### 1. 引用 IMA 知识库
**您说**："帮我查找供应商的联系方式"
**我会**：在 IMA 知识库的"供应商管理"中搜索相关信息

**您说**："之前记录的产品SKU信息是什么"
**我会**：在 IMA 知识库的"产品资料库"中查找

### 2. 引用本地记忆
**您说**："查看我今天的待办事项"
**我会**：读取本地 memory.md 中的记录

**您说**："我之前关于某个项目的想法是什么"
**我会**：读取本地 MEMORY.md 中的记录

### 3. 综合引用
**您说**："帮我生成一份供应商管理方案，参考之前的信息"
**我会**：
1. 从 IMA"供应商管理"检索联系方式、商务条款
2. 从本地 memory.md 检索相关想法和决策
3. 综合生成方案

## 支持的文件类型

- **文本文件**：.txt, .md, .docx, .doc
- **其他文件**：仅根据文件名进行路由判断

## 注意事项

1. 新文件会在1小时内的下一次自动化执行时被处理
2. 已处理的文件不会重复处理
3. 空文件或无法读取的文件仅记录文件名到本地
4. 所有处理结果都会记录日志，方便追溯
